# Think-aloud — coding book

This document defines the codes used to analyse the think-aloud
sessions. Two researchers code each transcript independently; we
report Cohen's $\kappa$ per code on a 20\% calibration sample and
target $\kappa \geq 0.70$ before moving to full coding.

## Axial structure

Codes are organised in four axes that mirror the BLE design:

| Axis | Codes |
|---|---|
| **A. Drawer affordance** | A1 noticed-drawer, A2 ignored-drawer, A3 opened-drawer, A4 toggled-cf |
| **B. Rule clause** | B1 paraphrased-rule, B2 questioned-rule, B3 rule-unclear, B4 rule-agreed, B5 rule-rejected |
| **C. Community signal** | C1 referenced-numbers, C2 mistrusted-numbers, C3 wanted-more-detail, C4 ignored-signal |
| **D. Decision moment** | D1 changed-intent-up, D2 changed-intent-down, D3 unchanged, D4 deferred |

Each code is binary at the **moment** (a 5–30 s segment) and per
participant.

## Open codes (emergent)

The four axes above are *seed* codes from the survey rationales. In
the first pass, the two coders also flag open codes for behaviour the
seeds don't capture. Open codes from the first three transcripts
become candidate axial codes for a calibration discussion before
full coding.

## Code definitions

### A1. noticed-drawer
The participant verbally or behaviorally indicates they noticed the
drawer (looked at it $> 1$ s, asked about it, named it). Does not
require opening.

### A2. ignored-drawer
The participant scrolled past a drawer without comment, look-pause,
or open. Coded per drawer instance.

### A3. opened-drawer
The participant interacted with the drawer affordance. Coded with
the boundary case ($b_1 \dots b_9$).

### A4. toggled-cf
The participant tapped the "Show what the label would say if the
rule did not apply" button. Note whether the counter-factual was a
surprise or expected.

### B1. paraphrased-rule
The participant restated the rule clause in their own words. Code
the paraphrase verbatim into the segment so we can later look at
which paraphrases recur.

### B2. questioned-rule
The participant articulated a worry about the rule (e.g., "but who
audits this?", "what if they change their mind?"). Cross-reference
with the survey's $C4$ \textsf{reversibility-concern} code.

### B3. rule-unclear
The participant failed to articulate what the rule meant, even when
prompted. Distinct from B2: B2 is questioning, B3 is not
understanding.

### B4. rule-agreed
The participant endorsed the rule ("yeah, that's fine").

### B5. rule-rejected
The participant disagreed with the rule.

### C1. referenced-numbers
The participant used the community percentages in their reasoning
("most people are OK with it so I'm OK").

### C2. mistrusted-numbers
The participant questioned where the numbers came from or rejected
them as decision input.

### C3. wanted-more-detail
The participant asked for more detail than the drawer provides ("OK
51\% accept, but who is the 49\%?").

### C4. ignored-signal
The participant noticed the community signal but did not use it.

### D1. changed-intent-up
After reading the drawer, the participant said they'd be more
likely to install.

### D2. changed-intent-down
After reading the drawer, the participant said they'd be less
likely to install.

### D3. unchanged
After reading the drawer, the participant's intent was the same as
before.

### D4. deferred
The participant chose not to make an install decision in the
session ("I'd want to think about it overnight").

## Reliability target
- Calibration sample: first 3 transcripts.
- Target: Cohen's $\kappa \geq 0.70$ on each code.
- If any code falls below 0.70 after the first calibration, the
  coders meet to refine the code definition and re-code that code on
  the calibration sample. Up to two calibration rounds before
  full coding begins.

## Quantification
Per-participant rates for the most frequent codes are reported as
descriptive statistics with bootstrap 95\% CIs. We do **not** run
inferential tests on the think-aloud counts (the design is
qualitative-led); we report counts and quotes to triangulate with
the vignette experiment results.
