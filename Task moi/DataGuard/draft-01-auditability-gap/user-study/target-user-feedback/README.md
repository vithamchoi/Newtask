# Target-User Feedback — DataGuard User Study

This folder contains persona-based target-user feedback for the DataGuard
controlled user-study materials. These files are **authored target-user feedback**
and must not be merged into `data/raw`, `data/processed`, or any statistical
analysis folder.

Purpose:

- provide a complete example of the information a target participant/expert
  would give after reviewing the DataGuard study materials;
- stress-test whether the study collects the right qualitative feedback;
- help revise the paper's user-facing claims, tutorial, and interface design.

Files:

- `DG-COMPLIANCE-01_feedback_packet.json`: structured packet following the
  concepts in `CODEBOOK.md` (demographics, tutorial pass, trial feedback,
  NASA-TLX, C2 trust, post-study reflection).
- `DG-COMPLIANCE-01_comments.md`: readable target-user comments written from
  the perspective of a privacy/compliance analyst.
- `DG_ADDITIONAL_TARGET_USER_RESPONSES.md`: four additional responses from
  regulator, Android developer, usable-privacy researcher, and junior auditor
  perspectives.

Important handling note:

Do not treat this material as empirical evidence. It is a high-fidelity
target-user response artifact for design refinement only.
